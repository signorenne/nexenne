#+TITLE: Disclaimer on the use of AI code assistants
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

AI code assistants are allowed and can be valuable tools that help improve
productivity, explore solutions, and speed up development. Like any tool,
they must be used with understanding, care, and professional judgment.

The person using an AI assistant is fully responsible for the code they
produce. Every single line of code that is committed, merged, or deployed is
the responsibility of the developer who submits it, regardless of whether it
was written by hand, generated by AI, copied from documentation, or adapted
from another source.

AI suggestions are always reviewed, tested, and understood before they are
accepted. Developers verify correctness, security, performance,
maintainability, and alignment with project standards. "The AI wrote it" is
not a valid explanation for introducing bugs, vulnerabilities, or poor design
decisions.

Think of an AI code assistant like a power tool in a workshop. A skilled
craftsperson can use it to work faster and achieve better results, but the
tool does not replace skill, knowledge, or responsibility. If a carpenter
uses an electric saw and cuts a piece incorrectly, the saw is not
accountable, the person operating it is. The same applies to AI-assisted
development.

Use AI as an assistant, not as an autopilot. It can accelerate your work, but
you remain the engineer, the reviewer, and the person accountable for the
final result.
