#+TITLE: nexenne modules
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

Every module is header-only C++23, dependency-light, and picked individually,
exposed as =nexenne::<name>= (both the CMake target and the namespace). A module
names its dependencies in =module.deps=, and the build discovers and orders them
automatically, so the set below grows as modules are ported. A module may only
depend on modules below it in the layering, which keeps the bottom of the stack
free-standing and the whole set acyclic.

Follow a module to its own index, which lists, per class it provides, a guide
(purpose, the idea, idiomatic usage) and a runnable example.

* Modules

- [[file:utility/README.org][utility]] (no dependencies) : the foundation layer, a grab-bag of 20 small,
  independent helpers: callables and visitors, scope-exit and RAII, bit and
  numeric and hashing helpers, strong distinct types, compile-time
  metaprogramming, and reflection. Everything else builds on it.
- [[file:container/README.org][container]] (needs =utility=) : header-only containers, fixed-capacity and
  dynamic, value and node-based: vectors (=static_vector=, =small_vector=,
  =stable_vector=), a ring buffer and deque, a bitset, a bag, a gap buffer, a
  heap, and bump arenas, all under one documented error policy.
- [[file:chrono/README.org][chrono]] (needs =utility=) : portable, clock-templated timing primitives:
  stopwatches, countdowns and polling timers, an alarm, a rate limiter, a frame
  timer, a scope timer and profiler, plus duration formatting and Hz/period
  helpers. The same code runs on a PC, an MCU, or a hand-advanced test clock.
- [[file:random/README.org][random]] (no dependencies) : seedable, reproducible RNG: the =xoshiro256ss=
  and =pcg32= engines, unbiased uniform draws, continuous and discrete
  distributions (normal, exponential, gamma, Poisson, discrete), and sampling
  (shuffle, reservoir, weighted choice). Deterministic across platforms, unlike
  =std::random='s implementation-defined distributions.
- [[file:benchmark/README.org][benchmark]] (needs =chrono=) : a micro-benchmark runner that auto-tunes the
  iteration count, takes multiple samples, and reports mean, median, stddev, and
  a noise gauge (=run=, =run_with_setup=, =compare=), plus the =do_not_optimize=
  and =clobber_memory= anti-dead-code primitives every honest benchmark needs. It
  reuses =chrono='s auto-scaling duration formatter for its ns/us/ms/s timings.
- [[file:serialization/README.org][serialization]] (needs =container=) : encoders and decoders under one
  =std::expected= error policy, never throwing and bounds-checking every byte: a
  JSON DOM, parser, and writers; a schema-driven binary =writer= / =reader=; the
  CBOR and MessagePack codecs; COBS framing; and a versioned envelope.
- [[file:filter/README.org][filter]] (no dependencies) : discrete-time digital filters for embedded signal
  conditioning, all sharing a =push= / =value= / =reset= surface: linear
  smoothers (=ema=, =sma=, =lowpass=, =highpass=, =biquad=, =butterworth=,
  =fir=), robust and adaptive filters (=median=, =kalman=, =complementary=,
  =lms=), control shapers (=slew=, =debounce=, =hysteresis=, =glitch=), and
  validation guards (=range_guard=, =rate_guard=, =validator=, =majority=,
  =stale_detector=).
- [[file:signal/README.org][signal]] (needs =container=, =utility=) : a header-only signal/slot system, a
  typed multicast callback channel with no virtual dispatch and no heap on the
  slot path. Priority-ordered =emit=, one-shot and lifetime-tracked
  subscriptions, a connect-only =sink= view, RAII =scoped_connection= /
  =emit_blocker=, and defined reentrancy (connect, disconnect, or destroy the
  signal from inside a slot). A fully allocation-free, fixed-capacity
  =static_signal= variant trades unbounded slots for zero heap.
- [[file:ecs/README.org][ecs]] (needs =container=, =signal=) : a sparse-set entity-component registry
  in the EnTT style, with no virtual dispatch. Recycle-safe =entity_id= handles,
  pointer-stable per-type component storage (so a view can add or remove
  components while iterating), =add= / =remove= / =get= / =has= / =patch=,
  multi-component =view= queries with an exclude list, and per-component
  =on_construct= / =on_update= / =on_destroy= lifecycle signals.
- [[file:algorithm/README.org][algorithm]] (needs =container=) : a toolbox of classic algorithms under one
  =std::expected= error policy: binary search, hashing (=fnv=, =murmur=,
  =xxhash=), checksums (a generic =crc= and the =modular_sum= / Fletcher
  family), encoding (=hex=, =base32=, =base64=, =url=, =cobs=), non-comparison
  sorts (=counting_sort=, =radix_sort=), string search and distance (=kmp=,
  =boyer_moore=, =z_function=, =levenshtein=, =aho_corasick=, =suffix_array=),
  numerical routines (=kahan_sum=, =online_stats=, =bisection=, =integration=,
  =interpolation=, =ode=, =fft=), and graph algorithms over =container::graph=
  (traversal, =topological_sort=, =tarjan_scc=, =kruskal_mst=, =dijkstra=,
  =bellman_ford=, =floyd_warshall=, =a_star=, =lca=).
- [[file:logging/README.org][logging]] (needs =container=) : a header-only leveled logging framework in the
  spdlog/quill style: a =level= enum, a =record= event, a virtual =sink=
  interface (=console=, =file=, =ring=, =multi=, =json=, =rotating_file=,
  =async=), a compile-time =config= policy selecting a sync or thread-backed
  async =manager=, a =logger= with =consteval= format strings that capture the
  source location, and the =LOG_*= macros gated at compile time. Plus a
  heap-free =stream_logger= for the MCU path with no alloc, thread, or queue.
- [[file:math/README.org][math]] (needs =utility=, =random=) : the numeric toolbox: fixed-size
  =vector<T, N>= and =matrix<T, N>=, unit quaternions and Euler angles for
  rotations, scalar utilities and constants, three-speed trigonometry, strong
  =angle= and Q-format =fixed= types, transform and projection matrices,
  parametric curves, and geometric samplers. Generic over the scalar type and
  =constexpr= where the language allows.

* Dependency layering

A module may only use the modules below it; the build refuses a cycle.

#+BEGIN_EXAMPLE
  ecs            entity-component registry (needs signal and container)
     |
     v
  benchmark      micro-benchmark runner
  serialization  JSON / CBOR / MessagePack / binary / COBS codecs
  signal         signal/slot multicast callbacks
  algorithm      search, hashing, sorting, string, numerical, graph algorithms
  logging        leveled logging with sinks, async manager, and macros
     |
     v
  container      vectors, maps, sets, queues, trees, graphs
  chrono         clock-templated timing primitives
     |
     v
  utility        the foundation, no dependencies

  math           vectors, matrices, quaternions (needs utility and random)
     |
     v
  random         seedable RNG (independent, no dependencies)
  filter         discrete-time digital filters (independent, no dependencies)
#+END_EXAMPLE

More modules join as they are ported, each declaring its place in the layering
through =module.deps=.
