#+TITLE: nexenne::container
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

Header-only C++23 containers, fixed-capacity and dynamic, value and
node-based, each picked individually and built on
[[file:../utility/README.org][nexenne::utility]]. They favour explicit capacity,
predictable addresses, and recoverable failure over the implicit growth and
throwing interfaces of the standard containers. Each entry below links to a
guide and a runnable example under
[[file:../../../examples/container/][examples/container/]]; everything lives in
=nexenne::container=.

*Error policy.* Failure is uniform and exception-free. Non-allocating operations
are =noexcept=. An operation that can fail at a documented boundary (pop from
empty, push when full, missing key) returns =result<T>=
(=std::expected<T, container_error>=) rather than throwing, so the caller decides
what to do. An operation whose only failure mode is allocation is =noexcept= and
terminates on =std::bad_alloc=; when you need to recover from that, pre-allocate
with =reserve= and check the boundary up front. The shared =container_error= enum
and the =result<T>= alias live in the [[file:error.org][error]] guide below.

*Choosing a category.* Pick by lifetime and access pattern. Reach for a /value
container/ when you want elements stored by value with a flat layout, choosing
=static_vector= or =small_vector= when the capacity is bounded and you want to
avoid the heap, =stable_vector= when references must stay valid across growth,
and =bag= or =bitset_dynamic= for set-like membership. Reach for a /sequence/
when you push and pop at the ends (=deque=) or edit around a cursor
(=gap_buffer=), a /priority queue/ (=heap=) when you want the largest element
first, and an /arena/ when many short-lived objects share one bump-allocated
region freed all at once.

* Foundation

- [[file:error.org][error]] : =container_error=, the =result<T>= alias, and =to_string=.

* Value containers

- [[file:static_vector.org][static_vector]] : fixed-capacity inline vector, no heap.
- [[file:small_vector.org][small_vector]] : inline for the first N, heap fallback beyond.
- [[file:ring_buffer.org][ring_buffer]] : fixed-capacity circular FIFO, fail or overwrite on full.
- [[file:stable_vector.org][stable_vector]] : append-only, element addresses never move.
- [[file:bitset_dynamic.org][bitset_dynamic]] : runtime-sized bitset, word-packed with popcount and scan.
- [[file:bag.org][bag]] : unordered multiset, O(1) swap-pop erase, allows duplicates.

* Sequences

- [[file:deque.org][deque]] : growable double-ended queue, contiguous power-of-two ring.
- [[file:gap_buffer.org][gap_buffer]] : text-editor buffer with a movable gap, O(1) edits at the cursor.

* Linked structures

- [[file:intrusive_list.org][intrusive_list]] : doubly-linked list with the links embedded in the element, O(1) erase.
- [[file:graph.org][graph]] : directed adjacency-list graph, dense vertex ids, optional edge payload.

* Ordered associative

- [[file:flat_set.org][flat_set]] : sorted-vector ordered set, O(log N) lookup, contiguous storage.
- [[file:flat_map.org][flat_map]] : sorted-vector ordered map, O(log N) lookup, contiguous storage.
- [[file:static_flat_map.org][static_flat_map]] : fixed-capacity inline ordered map, no heap, no growth.
- [[file:binary_tree.org][binary_tree]] : unbalanced ordered BST, node-per-element, in-order iteration.

* Hashed associative

- [[file:flat_hash_map.org][flat_hash_map]] : open-addressing hash map, linear probing, contiguous storage.
- [[file:flat_hash_set.org][flat_hash_set]] : open-addressing hash set, membership and dedup, contiguous storage.
- [[file:lru_cache.org][lru_cache]] : fixed-capacity cache, least-recently-used eviction, O(1) get/put.

* Sets and relations

- [[file:sparse_set.org][sparse_set]] : integer-key set, O(1) insert/erase/contains, dense iteration.
- [[file:dense_map.org][dense_map]] : integer-key map over a sparse_set, values packed for a dense walk.
- [[file:slot_map.org][slot_map]] : handle-keyed store, stable generation-tagged keys, O(1) lookup.
- [[file:union_find.org][union_find]] : disjoint-set partition, near-constant union and find.
- [[file:bimap.org][bimap]] : bidirectional map, unique on both sides, O(1) lookup either way.

* Priority queues

- [[file:heap.org][heap]] : binary heap over a vector, max-heap by default, configurable order.

* Arenas and pools

- [[file:linear_arena.org][linear_arena]] : inline-buffer bump allocator, O(1) reset.
- [[file:scratch_pad.org][scratch_pad]] : RAII checkpoint over an arena, rewinds on scope exit.
- [[file:object_pool.org][object_pool]] : fixed-capacity typed pool, O(1) acquire and release.
