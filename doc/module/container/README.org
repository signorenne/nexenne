#+TITLE: nexenne::container
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

Header-only C++23 containers, fixed-capacity and dynamic, value and
node-based, each picked individually and built on
[[file:../utility/README.org][nexenne::utility]]. Each entry below links to a
guide and a runnable example under
[[file:../../../examples/container/][examples/container/]].

*Error policy.* Non-allocating operations are =noexcept=. An operation that can
fail at a documented boundary (pop from empty, push when full, missing key)
returns =result<T>= (=std::expected<T, container_error>=), never an exception.
An operation whose only failure is allocation is =noexcept= and terminates on
=std::bad_alloc=; pre-allocate with =reserve= when recovery is needed.

* Foundation

- [[file:error.org][error]] : =container_error=, the =result<T>= alias, and =to_string=.

* Value containers

- [[file:static_vector.org][static_vector]] : fixed-capacity inline vector, no heap.
- [[file:small_vector.org][small_vector]] : inline for the first N, heap fallback beyond.
- [[file:ring_buffer.org][ring_buffer]] : fixed-capacity circular FIFO, fail or overwrite on full.
- [[file:stable_vector.org][stable_vector]] : append-only, element addresses never move.
- [[file:bitset_dynamic.org][bitset_dynamic]] : runtime-sized bitset, word-packed with popcount and scan.
