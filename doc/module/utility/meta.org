#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

* Purpose

=nexenne::utility::meta= bundles small compile-time helpers: =always_false_v=
for dependent static assertions, =type_name= for a diagnostic type spelling, and
=function_traits= for introspecting a callable's signature.

* The idea

Generic code often needs a callable's shape (its return type, arity, argument
types, and =noexcept=-ness) without the caller spelling them out.
=function_traits= recovers all of these for plain functions, function pointers,
member function pointers, and concrete-=operator()= functors, and the
=function_*= aliases (=function_return_t=, =function_arity_v=, =function_arg_t=,
=function_is_noexcept_v=, =function_args_t=) keep call sites terse.
=always_false_v= gives the unreachable branch of an =if constexpr= chain a
value-dependent =false=, so the diagnostic fires only for unhandled types. The
tradeoff: =type_name<T>()= relies on =__PRETTY_FUNCTION__= and returns an empty
view outside GCC/Clang, and a generic or overloaded =operator()= has no single
signature to introspect.

* How it works

The three helpers are independent:

- =always_false_v<Ts...>= is a variable template that is always =false= but whose
  value formally depends on =Ts=. That dependency is what makes
  =static_assert(always_false_v<T>, "...")= fire only when its =if constexpr=
  branch is actually instantiated, rather than at parse time.
- =type_name<T>()= reads the compiler's =__PRETTY_FUNCTION__= for this very
  function and slices out the spelling of =T= between a =T = = marker and the
  bounding bracket, coping with spaces, =::= qualifiers, commas inside template
  arguments, array types, and GCC's trailing =; = signature entries. It runs at
  compile time with no runtime cost.
- =function_traits<F>= is a trait whose specialisations cover plain function types,
  function pointers, and every cv- / ref- / =noexcept=-qualified member function
  pointer; the primary template handles a functor or lambda by delegating to the
  traits of its =operator()=. Each specialisation exposes a =return_type=, an
  =arguments= =type_list=, an =arity=, an indexed =arg_t=, and an =is_noexcept=
  flag. The =function_*= aliases are thin wrappers that strip a reference from =F=
  and read those members, so prefer the aliases over touching =function_traits=
  directly.

* When to use it

Reach for =function_traits= and the =function_*= aliases in generic code that must
adapt to a callable's signature: building a tuple of its argument types, checking
its arity, dispatching on its return type, or propagating its =noexcept=. Use
=always_false_v= to make the =else= of an =if constexpr= chain reject unsupported
types with a readable message, and =type_name<T>()= for diagnostic logging or test
output, never as a key or stable identifier.

The gotchas:

- =function_traits= needs *one concrete* =operator()=. A generic lambda (one with
  =auto= parameters) or any class with an overloaded or templated =operator()= has
  no single signature, so it does not introspect; give it a concrete signature
  first, for example by wrapping in a non-generic lambda.
- =type_name<T>()= is a *diagnostic label, not a stable name*. The spelling is
  compiler-dependent (it may carry an elaborated =struct= / =class= keyword,
  differ in qualification or whitespace) and on compilers other than GCC/Clang it
  returns an empty view, so never parse it or compare it for identity.
- =function_arg_t<F, I>= triggers a =static_assert= when =I= is out of range, so
  index within =function_arity_v<F>=.

* Usage

#+BEGIN_SRC cpp
auto compute(int a, double b) noexcept -> double;
using fn = decltype(&compute);

static_assert(nexenne::utility::function_arity_v<fn> == 2);
static_assert(std::is_same_v<nexenne::utility::function_return_t<fn>, double>);
static_assert(std::is_same_v<nexenne::utility::function_arg_t<fn, 0>, int>);
static_assert(nexenne::utility::function_is_noexcept_v<fn>);

constexpr auto label{nexenne::utility::type_name<double>()};  // contains "double"
#+END_SRC
