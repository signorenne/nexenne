#+TITLE: nexenne::math
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

The numeric toolbox of the library: fixed-size vectors and matrices, quaternions
and Euler angles for rotations, scalar utilities and constants, strong angle and
fixed-point types, and geometric samplers. It is the layer =geometry=, =physics=,
and =simulation= build on, but it is written to stand on its own on a
microcontroller too.

The spirit matches the rest of nexenne: everything is generic over the scalar
type, =constexpr= wherever the language allows, with no heap and no runtime
surprises. The types are *fixed-size*, =vector<T, N>= and =matrix<T, N>=, because
the dimension is known to the compiler, not the profiler.

* Scalar foundation

- [[file:constants.org][constants]] : pi, tau, e, the golden ratio, exact sin/cos at nice angles, and the
  degree/radian conversion factors, as =inline constexpr= variable templates that
  follow the caller's precision, plus unsuffixed =double= aliases. Forwards what
  =std::numbers= already has and adds what it lacks.
- [[file:scalar.org][scalar]] : the everyday single-number operations: selection (=clamp=,
  =saturate=, =move_toward=), interpolation (=lerp=, =smoothstep=, =remap=),
  float classification (=almost_equal=, =isnan=), rounding (=floor=, =round=,
  =fract=), and wrapping (=mod=, =repeat=, =wrap=, =ping_pong=). All =constexpr=
  and =noexcept=, with graphics-friendly conventions.
- [[file:power.org][power]] : =sqrt=, =inv_sqrt=, and =pow_int= with a =constexpr= path, plus the
  bit-trick approximations =fast_inv_sqrt=, =fast_exp=, and =fast_log= that trade
  a few digits for speed in the hot loop.
- [[file:trigonometry.org][trigonometry]] : sine, cosine, tangent and inverses at three speeds (libm
  =sincos=, =constexpr= polynomial =fast_*=, runtime =lut_*=), plus the
  short-way-around =angle_diff= and =lerp_angle=.

* Strong types for quantities

- [[file:angle.org][angle]] : =radians= and =degrees= as distinct strong types, so a unit mismatch
  is a compile error; carries the full algebra, typed conversions, and the
  =wrap_signed= / =wrap_unsigned= reductions.
- [[file:fixed.org][fixed]] : =fixed<Storage, FractionBits>= Q-format fixed-point for FPU-less or
  bit-exact-deterministic targets, with a doubly-wide intermediate for multiply
  and divide, and =q8_8= / =q16_16= / =q32_32= aliases.

* Vectors

- [[file:vector.org][vector]] : =vector<Value, N>=, the fixed-size vector with named =.x()= / =.y()=
  / =.z()= / =.w()= accessors and swizzles, the full element-wise algebra, and a
  tight contiguous =std::array= layout that the compiler auto-vectorizes to packed
  SSE.
- [[file:vector_algorithms.org][vector_algorithms]] : the operations on vectors: =dot=, =cross=, =length=,
  =normalize= and friends, =project= / =reject= / =reflect=, element-wise
  =lerp= / =clamp= / min / max, reductions, and =angle_between=.
- [[file:normalized.org][normalized]] : =normalized<Real, N>=, the strong wrapper that carries the
  unit-length guarantee in the type so direction-consuming APIs need not re-check.

* Matrices and transforms

- [[file:matrix.org][matrix]] : =matrix<Value, N>=, the column-major square matrix with =(row,col)=
  access, the full algebra, =transpose= / =determinant= / =inverse= / =trace=,
  and a column-combination multiply that compiles to packed SSE.
- [[file:projection.org][projection]] : the 4x4 =perspective= / =ortho= builders (plus the =_zo= Vulkan
  and Direct3D depth variants) that map view space to clip space.
- [[file:transform.org][transform]] : the affine builders (=translation=, =rotation=, =scale= in 2D and
  3D), the =look_at= view matrix, and =transform_point= / =transform_direction=.

* Curves and easing

- [[file:curve.org][curve]] : parametric =bezier_=, =catmull_rom=, and =hermite= evaluators with
  matching tangents, plus the =ease_smoothstep= / =ease_smootherstep= curves.

* Rotations

- [[file:quaternion.org][quaternion]] : =quaternion<Real>=, the unit quaternion for gimbal-lock-free 3D
  rotation, with composition, the cheap vector =rotate=, conversions to and from
  axis-angle / Euler / two-vectors / look-at / matrices, and =slerp= / =nlerp=.
- [[file:slerp_variants.org][slerp_variants]] : the explicitly-named interpolation choices =nlerp_plain= /
  =nlerp_short= / =slerp_short=.
- [[file:euler.org][euler]] : the =euler_order= enum, the =euler_angles= carrier, and conversion to a
  quaternion under any of the six orders (plus the aerospace =from_ypr=).

* Random sampling

- [[file:random.org][random]] : geometric samplers on the nexenne::random engines: =unit_vector2= /
  =unit_vector3= directions, =point_in_unit_disc= / =point_in_unit_ball=, and
  =random_angle=.

* Standard-library integration

- [[file:hash.org][hash]] : =std::hash= specializations so =vector=, =matrix=, and =quaternion=
  work as =unordered_= keys.
- [[file:format.org][format]] : =to_string=, =operator<<=, and =std::formatter= for =vector=,
  =matrix=, =quaternion=, and =normalized=.

* Headers without an article

- =math.hpp= : the umbrella header; includes the whole module.
- =concepts.hpp= : the internal constraints (=arithmetic=, =signed_arithmetic=,
  =affine_point=) that keep the templates honest.
- =error.hpp= : the =math_error= enum and the =result<T>= alias
  (=std::expected<T, math_error>=) returned by the fallible operations.

* Design pillars

** Fixed-size, generic over the scalar
   =vector<T, N>= and =matrix<T, N>= have a compile-time size and are generic over
   the scalar type, so the same code serves =float=, =double=, and fixed-point.
   No allocations, no runtime dimensions.

** constexpr where it counts
   Much of the algebra is usable in constant evaluation, so transforms known at
   compile time cost nothing at runtime; this is checked by =static_assert= tests.

** Strong types for quantities
   An =angle= is not just a =double=, and a =normalized= vector is not just a
   =vector=: the module uses the type system to rule out the classic mistakes,
   like adding degrees to radians or forgetting to normalize.

** Rotations without gimbal lock
   For 3D rotation the reference representation is the unit quaternion; Euler
   angles stay for human input, with the usual cautions. Interpolation goes
   through SLERP.

** Never throws
   Every operation is =noexcept=. Operations with a real runtime failure mode
   (singular matrix, zero-length vector) return =result<T>=; pure infallible
   operations return their natural type directly. See [[file:../../../modules/math/include/nexenne/math/error.hpp][=error.hpp=]].

* Dependencies

=math= depends on two library modules:

- [[file:../utility/README.org][utility]] : the base concepts, hashing, and strong types.
- [[file:../random/README.org][random]] : the engines the geometric samplers draw from.

In turn =math= is the base of =geometry=, =physics=, and =simulation=.
