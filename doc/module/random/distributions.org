* Purpose

Sampler objects for the common non-uniform distributions: =normal=,
=exponential=, =gamma=, =poisson= (continuous and count), and a =discrete=
weighted-index distribution. Each is a small object holding its parameters, with a
=sample(engine)= that draws one value from any =rng_engine=. Unlike
=std::random='s distributions, the output is reproducible across platforms.

* The idea

A distribution sampler turns uniform randomness into a shaped one. There are three
classic strategies, and each sampler here uses whichever fits its shape:

- *Inverse transform*: if =F= is the cumulative distribution function (CDF) and
  =U= is uniform on =[0, 1)=, then =F^{-1}(U)= has distribution =F=. Cheap when
  =F^{-1}= has a closed form (exponential, discrete).
- *Rejection sampling*: draw from an easy proposal that dominates the target, keep
  a sample with a probability that corrects the shape, redraw otherwise. Used when
  no closed-form inverse exists (gamma).
- *Transformation*: map several uniforms through a function whose output already
  has the target shape (Box-Muller for the normal).

The objects cache what they can (a precomputed factor, a spare normal variate) so
repeated sampling is cheap. For the deep theory, Devroye's freely available book
(see Further reading) is the standard reference for all of these.

* How they work

** normal

=normal_distribution<T>(mean, stddev)= uses the Box-Muller transform (1958). Its
trick: a 2-D standard normal is *rotationally symmetric*, so in polar coordinates
the angle is uniform on =[0, 2*pi)= and the squared radius is exponential. So draw
two uniforms =U1, U2=, set radius =sqrt(-2 ln U1)= and angle =2*pi*U2=, and the two
Cartesian components =radius*cos(angle)= and =radius*sin(angle)= are *two
independent* standard normals. The sampler returns one and caches the second for
the next call, halving the per-sample cost (two transcendental calls amortise over
two samples). The standard normal is then scaled and shifted to =N(mean,
stddev^2)=. A negative stddev is folded to its absolute value.

** exponential

=exponential_distribution<T>(rate)= inverts the CDF: =-log(U) / rate= for a
uniform =U= in =(0, 1]= (the zero is rejected so the logarithm stays finite). One
=log= per sample; the mean is =1 / rate=.

** gamma

=gamma_distribution<T>(shape, scale)= uses the Marsaglia-Tsang method (2000) for
shape =>= 1: propose a standard normal =x=, form =v = (1 + c*x)^3= for a constant
=c= depending on the shape, and accept =d*v= with a probability that reshapes the
normal into a gamma. The acceptance test has a cheap polynomial "squeeze" that
short-circuits most accepts before the expensive logarithm, so it averages close
to one normal draw per sample. For shape =< 1= it uses Marsaglia's boosting
identity: a =Gamma(shape)= variate equals a =Gamma(shape + 1)= variate times
=U^(1/shape)=, so it samples the easier =shape + 1= case and scales down. The mean
is =shape * scale=; gamma generalises both the exponential (shape 1) and the
chi-squared.

** poisson

=poisson_distribution<T>(lambda)= uses Knuth's product-of-uniforms method for a
small mean (precomputing =exp(-lambda)=) and switches to a normal approximation,
rounded to the nearest non-negative integer, for a large mean where Knuth's loop
would be slow. The mean is =lambda=.

** discrete

=discrete_distribution<T>(weights)= builds a cumulative-weight table once, then
each =sample= draws a uniform point in =[0, total)= and binary-searches the table,
so selection is =O(log k)= and proportional to the weights. Zero and negative
weights are clamped out; an empty or all-zero distribution returns an
out-of-range index as its "no choice" signal.

#+BEGIN_EXAMPLE
discrete weights {1, 3, 6}  ->  index 0 at 10%, index 1 at 30%, index 2 at 60%
#+END_EXAMPLE

* Complexity

| distribution        | per-sample cost                        |
|---------------------+----------------------------------------|
| normal              | O(1) amortised (cached spare variate)  |
| exponential         | O(1), one log                          |
| gamma               | O(1) expected (rejection)              |
| poisson             | O(lambda) small / O(1) large           |
| discrete            | O(log k) binary search                 |

* When to use them

- =normal= for measurement noise, jitter, and any bell-shaped quantity.
- =exponential= for waiting times between independent events.
- =gamma= for positive skewed quantities and as a building block (it generalises
  exponential and chi-squared).
- =poisson= for counts of independent events in a fixed interval (arrivals,
  decays).
- =discrete= for a weighted pick among labelled outcomes (loot tables, weighted
  menus); for a one-off weighted index without building an object, see
  [[file:sampling.org][weighted_choice]].

Things to keep in mind:

- These are statistical: a *fixed seed* makes the stream reproducible, but any one
  sample is still a draw; check means over many samples, not single values.
- Parameters are taken as given (stddev folded to absolute, weights clamped
  non-negative); pass sensible values.

* Usage

#+BEGIN_SRC cpp
namespace rnd = nexenne::random;
rnd::pcg32 g{seed, 1};

rnd::normal_distribution<double> noise{0.0, 1.0};
auto const n{noise.sample(g)};               // ~ N(0, 1)

rnd::discrete_distribution<double> loot{std::array{70.0, 25.0, 5.0}};
auto const drop{loot.sample(g)};             // 0, 1, or 2 by weight
#+END_SRC

- Luc Devroye, "Non-Uniform Random Variate Generation" (Springer, 1986), the
  comprehensive reference for every method here, free from the author:
  http://www.nrbook.com/devroye/
- G. E. P. Box and Mervin Muller, "A Note on the Generation of Random Normal
  Deviates" (Annals of Mathematical Statistics, 1958), the original normal
  transform: https://doi.org/10.1214/aoms/1177706645
- George Marsaglia and Wai Wan Tsang, "A Simple Method for Generating Gamma
  Variables" (ACM TOMS, 2000), the gamma sampler used here:
  https://doi.org/10.1145/358407.358414
- Donald Knuth, "The Art of Computer Programming, Vol. 2: Seminumerical
  Algorithms" (3rd ed., 1997), section 3.4.1 for the Poisson and the inverse and
  rejection methods.
- A. J. Walker, "An Efficient Method for Generating Discrete Random Variables with
  General Distributions" (ACM TOMS, 1977), and Michael Vose's =O(n)= setup
  variant (1991), the alias method, an =O(1)=-per-draw alternative to the
  cumulative-table =discrete_distribution= when weights are fixed:
  https://doi.org/10.1145/355744.355749
- Wikipedia overviews, good starting points with diagrams: "Inverse transform
  sampling", "Rejection sampling", "Box-Muller transform", "Gamma distribution",
  "Poisson distribution".
