How the project works and how to use it: a few lines per topic, each linking to
the real files. Start here, follow the links. Per-module usage guides live under
[[file:module/][module/]]; setup and tooling docs live under [[file:setup/][setup/]].

* Modules

- [[file:module/utility/README.org][utility]] : small, focused helpers (callables, RAII, bit/numeric, strong
  types, metaprogramming, reflection). One guide and one runnable example per
  class.
- [[file:module/container/README.org][container]] : fixed-capacity and dynamic containers (vectors, maps, sets,
  queues, trees, graphs), built on =utility=.

* Setup and infrastructure

- [[file:setup/build-system.org][Build system]] : root CMake, options, presets.
- [[file:setup/modules.org][Modules]] : layout, =module.deps=, discovery, kinds, standalone builds.
- [[file:setup/cmake-helpers.org][CMake helpers]] : what each file under =cmake/= does.
- [[file:setup/install.org][Install and consume]] : =find_package= / =add_subdirectory= / =FetchContent=.
- [[file:setup/testing.org][Testing]] : doctest and the =nexenne_tests= target.
- [[file:setup/hooks.org][Git hooks]] : the pre-commit and pre-push gates.
- [[file:setup/ci.org][CI]] : the GitHub Actions jobs.
- [[file:setup/api-docs.org][API docs]] : Doxygen, and what the docs build enforces.
- [[file:setup/conventions.org][Conventions]] : code style, naming, file headers, commits.
- [[file:setup/tools.org][Tools and tech]] : every external tool used, what for, where to learn more.
