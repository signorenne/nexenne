#+TITLE: Git hooks
#+OPTIONS: toc:nil num:nil ^:nil
#+STARTUP: showall

Two local gates live in [[file:../../.githooks/][.githooks/]] (versioned in the
repo, not in =.git/hooks/=). Enable them once per clone by pointing git at the
directory:

#+BEGIN_SRC sh
git config core.hooksPath .githooks
#+END_SRC

* The hooks

- [[file:../../.githooks/pre-commit][pre-commit]] (on commit) : the staged
  =.hpp= / =.cpp= files (added, copied, or modified) must be clang-format clean.
  It compares =clang-format <file>= against the file and lists any that differ,
  telling you to run =clang-format -i <file>=. Fast, so it runs on every commit.
  If =clang-format= is not on =PATH= the check is skipped (it does not fail).
- [[file:../../.githooks/pre-push][pre-push]] (on push) : the full gate. Runs
  =cmake --preset ci=, =cmake --build --preset ci=, and
  =cmake --build --preset ci --target nexenne_tests=, so the push is blocked
  unless the project configures, builds with warnings as errors, and every
  doctest suite passes.

Format runs on every commit to stay cheap; the heavier build + test runs once,
on push.

* Skipping

Bypass either hook for a single git command by prefixing it with
=NEXENNE_SKIP_HOOK=1=:

#+BEGIN_SRC sh
NEXENNE_SKIP_HOOK=1 git commit -m "wip"
NEXENNE_SKIP_HOOK=1 git push
#+END_SRC

The same checks run in [[file:ci.org][CI]] (=format=, =build-test=), so a skipped
hook is still caught on the server.
