How the project is built, tested, documented, and shipped: reference docs for
the build system and tooling, kept separate from the per-class API guides (for
the libraries themselves see [[file:../module/README.org][Modules]], or the
guides under each =modules/<name>/doc/=). One topic per file, grouped below into
the build pipeline, the quality gates that guard every change, and the
conventions and external tools the project leans on. New here? Run the quick
start, then read /Build system/ for the whole picture, /Modules/ to add a
module, or /Install and consume/ to use nexenne in your own project.

* Quick start

#+BEGIN_SRC sh
git config core.hooksPath .githooks     # enable the local format/test gates
cmake --preset dev                       # configure (Debug, tests + examples on)
cmake --build --preset dev               # build everything enabled
cmake --build --preset dev --target nexenne_tests   # build and run all suites
#+END_SRC

The presets are =dev=, =release=, =asan=, =ci= (configure + build) plus the
=tests= build preset. The umbrella library is =nexenne::all=; each module is
=nexenne::<name>=. See [[file:build-system.org][Build system]] for the full picture.
